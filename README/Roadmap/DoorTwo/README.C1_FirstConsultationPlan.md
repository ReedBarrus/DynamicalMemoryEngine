1. Why this note exists

The project now has its first live C1 object stored in canon/.

That creates a new practical question:

Can the first live C1 object be consulted lawfully under real reuse pressure, and can the result remain governable if the new evidence is weaker, mixed, or contradictory?

This note exists to define the first consultation before the next implementation/review pass.

The immediate goal is not breadth.
The immediate goal is to prove one live canon object can be:

consulted,
bounded,
challenged if needed,
and kept honest under same-family reuse.
2. Consultation target
Live C1 object
canon/C1_BASELINE_SINE400_001.json
Current role

This object is a narrow same-family, same-lens baseline comparison anchor.

It is not:

a universal audio canon,
a semantic identity object,
a prediction object,
or a cross-family regime classifier.

Its current role is a trusted reference peg only.

3. Consultation scenario

The first consultation should use a same-family reuse scenario.

Recommended scenario:

record a new audio cohort in the same practical family as the original daw_mic_sine_400hz material
preserve the same declared medium FFT/Hann lens
preserve the same structural baseline / perturbation / return style if possible
use the live C1 only as a comparison anchor, not as a reconstruction authority

The first consultation should remain as close as practical to the original family because the point is not to stretch the canon object.
The point is to test whether the first active C1 can support lawful reuse under its declared scope.

4. Declared consultation question

The core question is:

Under the same declared lens and same family scope, does the new run resemble the trusted baseline anchor strongly enough that the current C1 remains a lawful comparison peg?

This is a comparison / recognition question.

It is not:

a truth question,
a prediction question,
a semantic identity question,
or a cross-family generalization question.
5. Allowed consultation use

The first consultation should request only uses already authorized by the live C1 object.

The intended allowed use is limited to:

same-family baseline comparison
same-family perturbation-vs-baseline comparison
same-family narrow return-vs-perturb reference
bounded review-anchor consultation under the same declared lens

Any request outside that list should fail closed.

6. Required declared conditions

The first consultation should remain lawful only if all of the following are explicit:

A. Source-family posture

The new run must remain inside the same practical source family as the live C1 object.

B. Lens posture

The new run must use the same declared medium FFT/Hann lens family.

C. Scope posture

The consultation must remain same-family and review-bounded.

D. Status posture

The live C1 object must remain in a consultable status:

promoted
contested
or narrowed
E. Handoff posture

The handoff seam must preserve:

effective scope visibility
allowed-use visibility
fail-closed denial on mismatch
7. What Claude should test

The next implementation/review pass should test one bounded consultation scenario and one governance-pressure scenario.

Test B — challenge-pressure observation

Claude should evaluate whether the new run pressures the current challenge trigger for Packet 1 strongly enough to justify:

8. Expected outcomes

The first consultation pass should be prepared to return one of the following bounded outcomes.

Outcome 1 — keep promoted

Use when:

the new same-family run remains sufficiently aligned with the baseline-anchor role,
no meaningful challenge trigger is crossed,
and the consultation seam behaves as intended.
Outcome 2 — annotate for review only

Use when:

consultation is lawful,
but the new run shows enough drift or ambiguity that the anchor should be re-read carefully,
without yet contesting the canon.

This is a review note, not a status change.

Outcome 3 — recommend contested review

Use when:

consultation is lawful,
but the new run appears to materially challenge the current anchor role under the same family and same lens,
and the challenge trigger is plausibly crossed.

This should recommend review.
It should not silently change canon status inside the same patch.

Outcome 4 — deny consultation

Use when:

source family does not match,
lens does not match,
allowed use does not match,
or required fields are missing.

This is a lawful fail-closed result.
