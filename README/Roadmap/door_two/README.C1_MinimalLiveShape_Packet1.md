# 9. Shape behavior rules
Rule 1 — claim text outranks summary text

promoted_claim is authoritative for the live C1 object.
claim_summary is convenience only.

Rule 2 — scope fields are mandatory

No live C1 object exists without explicit scope fields.

Rule 3 — allowed use must be explicit

If allowed_use is absent, the C1 object must not be runtime-consultable.

Rule 4 — forbidden use must be explicit

If forbidden_use is absent, the C1 object must fail closed.

Rule 5 — handoff enablement must be explicit

A promoted object without runtime_handoff_enabled: true is not yet live-consultable.

Rule 6 — support refs are not promoted by inclusion

Support references remain supporting evidence only.
