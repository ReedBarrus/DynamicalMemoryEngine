0. Purpose

	This document defines the lawful integration boundary between:

		DME (Dynamical Memory Engine)
			→ structural memory substrate (Door One runtime)
		RSI (Recursive Semantic Infrastructure)
			→ semantic constraint / viability substrate

	The purpose of this bridge is to:

		enable controlled translation between structural dynamics and semantic structure
		prevent premature ontology fusion
		preserve constitutional authority boundaries
		allow targeted experimental coupling

1. Constitutional Position

	1.1 Layer Alignment

		DME defines a strict progression:

			measurement → structure → runtime memory → recognition → canon → prediction → agency

		RSI operates primarily in:

			recognition space
			pre-canonical semantic structuring

		Therefore:

			RSI is downstream-interpretable, but not upstream-authoritative

	1.2 Foundational Boundary

		Door One runtime:
			does not produce truth
			does not produce meaning
			produces structural memory and observable dynamics
		RSI:
			does not rewrite substrate
			does not override runtime outputs
			produces semantic interpretations of structural persistence

	1.3 Core Rule

		Structural memory is authoritative for what occurred
		Semantic structure is interpretive for what persists as meaning-like structure

	2. Shared Invariants (Hard Alignment)

		These are lawful correspondences between DME and RSI.

	2.1 Recurrence
		DME: repeated windowing / trajectory continuity / segment evolution
		RSI: persistence across recurrence layers

	✔ Invariant:

		Recurrence is the primary carrier of identity in both systems

	2.2 Basin Formation (Structural ↔ Semantic)

		DME:
			structural neighborhoods
			segment continuity
			trajectory clustering
		RSI:
			semantic basins
			viability concentration regions

		✔ Invariant:

			Basins emerge from persistent structure under recurrence

	2.3 Persistence

			segment overlap

		RSI:

		RSI:

3. Non-Equivalences (Critical Safeguards)

	These prevent hypomanic fusion errors.

	3.1 Structure ≠ Meaning
		DME basin ≠ semantic claim
		structural clustering ≠ interpretation

		RSI must earn semantic interpretation via coherence

	3.2 Runtime ≠ Canon
		Door One outputs are:
			observable
			replayable
			auditable

		They are not:

			truth
			meaning
			validated knowledge
			
	3.3 Query ≠ Understanding
		QueryOp:
			selects
			filters
			retrieves

		RSI:

			interprets persistence
			evaluates coherence

	3.4 Dimensions ≠ Features
		DME:
			operates on signal features / transforms
		RSI:
			dimensions = coherence-stabilized transformation directions

		Features do not automatically become dimensions

	3.5 Amplitude ≠ Viability
		signal strength ≠ semantic viability
		frequency content ≠ coherence
