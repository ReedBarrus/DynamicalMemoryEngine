## 2. Why this note exists

Recent replay and real-source probe work clarified an important limitation in the current experimental language.

The current baseline / perturbation / return triad remains useful as:

- experiment choreography,
- capstone design,
- and replay-comparison scaffolding.

But it is too coarse to act as the primitive structural vocabulary for continuous real streams.

In particular:

- synthetic replay families supported exchange-regime recovery under tightly controlled conditions,
- first real-source sliced cohorts instead showed profile-geometry recovery under non-oscillatory drift,
- continuous unlabeled master-stream probing then recovered provisional candidate boundaries and return-like structure without relying on hand labels.

These findings suggest that Door One now needs a richer **structural transition vocabulary** for read-side interpretation.

The purpose is not to replace evidence with language.
The purpose is to stop a simple experimental triad from hardening into hidden ontology.

Baseline / perturbation / return remains an **experiment scaffold**.

## 4. Structural transition vocabulary (candidate classes)

The following candidate classes are now allowed as **read-side structural interpretation vocabulary**:

- **dwell**  
  relatively stable persistence under one structural profile

- **drift**  
  gradual movement in profile space without a sharp regime break

- **ingress**  
  entry into a new structural region or emerging profile family

- **coupling**  
  increased shared structural participation between previously separated components or bands

- **decoupling**  
  reduction in previously coupled structural participation

- **distortion**  
  profile deformation relative to a prior local recurrence class without full rupture

- **rupture**  
  sharp structural discontinuity or strong candidate boundary event

- **collapse**  
  contraction from a richer or more distributed profile into a narrower or reduced one

- **re-entry**  
  later return-like convergence toward an earlier structural region

- **branching**  
  divergence into multiple candidate continuation families or ambiguous next-state structure

These are interpretation classes only.

They do **not** imply:
- cause,
- semantic event labels,
- identity truth,
- canonical state labels,
- or promotion.

---

Coupling and decoupling should be used conservatively.

They are preferred only where the active lens supports a credible change in shared structural participation or coordinated profile behavior.

Where that support is weak, Door One should prefer:
- `drift`
- `ingress`
- `unresolved`

over premature coupling language.

## Locality rule

Candidate transition classes are local read-side interpretations.

A single stream may contain multiple candidate transition rows, overlapping ambiguity regions, and partially conflicting local readings.

Door One should not force one dominant transition story for a full file or cohort merely because local candidate classes are available.

## 6. Placement rule

This vocabulary belongs:

- downstream of ingest,
- downstream of raw structural runtime artifacts,
- downstream of provenance-preserving measurement handling,
- and inside read-side interpretation / probe surfaces.

It does **not** belong:

- inside the ingest contract,
- as a hidden adapter semantic layer,
- as a BasinOp rewrite,
- as an AnomalyOp expansion that dominates all change interpretation,
- or as a runtime truth surface.

Reason:

Door One ingest must remain measurement-adjacent and provenance-preserving.
Scripts and inspection surfaces may compute bounded structural interpretations, but they must not silently redefine runtime meaning.

---

## 7. Relation to anomaly

Anomaly remains one useful deviation surface.

It is not the whole transition vocabulary.

This note exists in part to prevent the project from collapsing all meaningful transition structure into “anomaly.”

Examples:

- lawful dwell is not anomaly,
- lawful return / re-entry is not anomaly,
- gradual drift is not necessarily anomaly,
- recurrence restructuring may be boundary-rich without being anomalous in the weak everyday sense.

Anomaly may remain one contributing signal among several.
It should not dominate the entire read-side segmentation language by default.

---

## 8. Relation to basin-facing diagnostics

This vocabulary may sit adjacent to basin-facing diagnostics, but it does not redefine basin meaning.

Basin diagnostics remain governed by their own bounded posture and explanatory ladder.

A transition class such as rupture, re-entry, or branching may be useful when describing how a stream moves through profile space over time.

That does **not** make a basin into ontology or a transition into attractor truth.

---

## 9. Lens discipline

Every use of this vocabulary must remain declared-lens dependent.

At minimum, outputs should preserve:

- source identity
- stream identity
- modality / channel
- sample-rate posture
- decimation if any
- window family
- hop policy
- band partition
- slice or macro-segment rule if any
- derived-vs-durable posture

A transition label is only meaningful relative to the analysis lens that produced it.

Changing the lens without declaring it can create false transition narratives.

---

## Support-basis rule

Where practical, each emitted candidate transition class should preserve a compact support basis indicating which read-side surfaces justified the label.

Examples include:

- `boundary_peak`
- `profile_shift`
- `recurrence_match`
- `anomaly_density_change`
- `basin_split_observed`
- `weak_support_only`

This support basis remains structural and auditable.
It does not convert the label into truth.
Its purpose is to preserve interpretive accountability.

## 10. Ambiguity rule

If evidence does not support a stable transition class, the system must say so.

Allowed ambiguity postures include:

- `unresolved`
- `weak_boundary_signal`
- `continuous_drift`
- `ambiguous_reentry`
- `insufficient_support`

Ambiguity is an acceptable result.

Forcing clean transition stories where support is weak is not.

Observed unsupported / excluded classes for this cohort:

## Distortion audit relation

This vocabulary should be reviewed whenever a read-side surface begins compressing distinct structural realities into one transition label strongly enough that downstream interpretation becomes misleading.

In such cases, Door One should prefer:
- clarification of language,
- unresolved posture,
- or a richer derived read-side probe,

before modifying upstream runtime layers.

## Mechanism flattening note (400hz master.wav perturb)

The current transition vocabulary is intentionally coarse and read-side.

A concrete flattening is now observed in real-source work:

- broadband redistribution events
- narrow-band spectral concentration events

may both be labeled `rupture` when they exceed the same structural boundary threshold.

This does not invalidate the `rupture` class.

It means the class currently names boundary severity better than boundary mechanism.

Until a separate read-side concentration probe is added, `rupture` should be read as:

- strong structural discontinuity,
not:
- a full description of how the energy shifted.
