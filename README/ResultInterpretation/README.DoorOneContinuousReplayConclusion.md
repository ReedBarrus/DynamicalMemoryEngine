# Door One Continuous Replay Loop v0 — Result Note

## Status

This note records the outcome of the first bounded Door One continuous replay capstone.

It is a supporting experiment note.
It does not mint canon, truth, ontology, or promotion.

---

## Experiment posture

This experiment used:

- one source family,
- one shared stream lineage,
- one declared lens family,
- one perturbation axis (`continuity`),
- one bounded 9-run cohort:
  - 3 baseline runs,
  - 3 perturbation runs,
  - 3 return runs.

The source was a controlled DAW-generated sine-wave family rendered as bounded WAV files.
The perturbation condition inserted a continuity break into the otherwise stable signal.

---

## Lens

The cohort was run under the established tighter audio lens:

- `Fs_target = 2400`
- `base_window_N = 1024`
- `hop_N = 512`

This lens was carried forward from the earlier audio slice capstone because the coarser whole-file support had previously under-resolved short perturbation structure.

---

## Cohort outcome

The experiment succeeded as a bounded Door One continuous replay capstone.

All 9 runs completed successfully under one shared stream lineage, with consistent sample count and duration across the cohort.

Observed structural pattern:

- all three baseline runs showed:
  - `state_count = 33`
  - `basin_count = 1`
  - `segment_count = 1`

- all three continuity-break runs showed:
  - `state_count = 31`
  - `basin_count = 5`
  - `segment_count = 3`

- all three return runs reverted to:
  - `state_count = 33`
  - `basin_count = 1`
  - `segment_count = 1`

This indicates that Door One did not merely detect generic difference.
It produced a stable repeated distinction between:

- a smooth single-regime continuity condition,
- a structurally broken continuity condition with multiple basins and segments,
- and a return condition that collapsed back to the baseline regime rather than remaining drifted.

---

## Cross-run reading

The cross-run report strengthens this result.

All 9 runs were compared inside one stream scope rather than being split into separate stream families.
This was the correct identity posture for the experiment because the goal was cross-phase comparison within one source family.

Pairwise comparison patterns were coherent:

- baseline-to-baseline similarity: high
- baseline-to-return similarity: high
- baseline-to-continuity-break similarity: low
- continuity-break runs consistently showed elevated:
  - basin count,
  - segment count,
  - transitions,
  - re-entries

relative to baseline and return.

The per-run signatures were also consistent with the designed perturbation:

- baseline and return:
  - `sticky`
  - `smooth`
  - low transition density
  - high attention concentration / persistence

- continuity-break:
  - `hopping`
  - `novelty-driven`
  - medium transition density
  - medium attention concentration / volatility
  - absent pre-commitment

These labels remain runtime/read-side summaries, not settled ontology.
However, they are supported by the measured structural deltas.

---

## Replay / retention result

The experiment also succeeded at the replay-support layer.

The cohort preserved:

- bounded per-run execution,
- durable receipt-backed lineage,
- cross-run session accumulation,
- replay-support availability across the full 9-run cohort.

This is sufficient to treat the cohort as a Door One reference packet for later comparison.

This does **not** imply canon, promotion, or truth uplift.
It remains a bounded runtime/replay artifact below canon.

---

## Supported claim

This experiment supports the following bounded claim:

**Door One can, under one declared ingest/runtime lens, distinguish a stable continuity regime from a deliberate continuity-break regime and recognize a return to the original regime across a 9-run bounded cohort while preserving one shared stream lineage and replay-supporting run/session structure.**

This is stronger than the earlier file-slice result because it is now:

- repeated,
- cohort-based,
- cross-run,
- retention-aware.

---

## Non-claims

This experiment does not yet establish:

- robustness to contamination or white noise,
- direct live-stream capture behavior,
- multi-scale invariance,
- transform-family robustness,
- simultaneous multi-input processing,
- canonical truth about the source.

It remains a bounded Door One experiment.
